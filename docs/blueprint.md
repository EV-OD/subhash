# **App Name**: Harmony Touchpoint

## Core Features:

- Mission Snapshot: Showcase the company's mission and values on the homepage.
- Services Overview: Present detailed information about provided home care services (personal care, companionship, supportive care).
- Testimonials: Provide testimonials from clients/families to build trust.
- Keystatic CMS: Enable content updates, and blog posts via a visual admin panel.
- Accessible design: Make all content legible, readable, and compatible with screen readers.
- Scalable architecture: Use the Nextjs framework in a way that new services can be added easily as the business grows.
- Contact Page: Provide forms, click-to-call, and embedded Google map functionality.
- Blog System: Create a blog with post listing and individual article pages rendered from Markdown via Keystatic CMS.
- Centralized Constants: Store color codes, text content, navigation links, and global CTA text in a `constants.ts` file for consistency.
- Modular File Structure: Organize code into folders such as `/components`, `/content`, `/utils`, `/styles`, and `/config` to ensure maintainability and scalability.
- SEO Optimization: Include meta tags, descriptive page titles, alt text for images, and clean URLs to help improve search engine visibility.
- Vercel Deployment: Host the site on Vercel's free tier with support for server-side rendering, CMS, and GitHub auto-deployments.

## Style Guidelines:

- Primary color: Gold-Brown (#AA8635) to evoke a warm feeling.
- Background color: Cream (#F6F2EA) for a calm backdrop.
- Accent color: Teal (#35AA86) for interactive elements.
- Text color: Dark Brown (#33280F) for headings.
- Heading font: 'Lora', a transitional serif, to convey an intellectual feel.
- Body font: 'Inter', a grotesque sans-serif font, for a modern and objective feel.
- Use simple, elegant icons that represent services. For instance, use heart symbols to convey personal connection. All iconography should aim to reinforce the feelings of trust and empathy.
- Maintain a clean and intuitive layout that puts important content in key locations so that users can find what they need as soon as they need it. Employ a grid layout and whitespace to direct attention to important areas.
- Use responsive design principles to ensure optimal usability on mobile, tablet, and desktop devices.
- Ensure sufficient color contrast and font sizes to meet WCAG 2.1 AA accessibility standards.
- Include visible focus styles for keyboard navigation to enhance accessibility.
- Test with screen readers and keyboard-only navigation throughout development.
